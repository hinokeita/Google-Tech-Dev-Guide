問題0：本番環境にバグがある。
  ・まずは、問題を抱えているという事実を認めることから。
  ・大量のテストをしてもバグは発生してしまう。

問題1：どこからバグが発生したか分からない。
  ・自分がいるスレッド、クラス、重要度の高いメソッドなどのように細かい単位でログを出力する。
  ・UUIDを生成して、ログに書き出す。

問題2：補足されなかった例外エラーに対する情報がない。
　・例外でキャッチできないとコンテキストのほとんどが失われる。
　・TLS(スレッドローカルストレージ)を使用するとスレッド自体に変数を格納できる。
　・下記のようにTLSを使用すると例外がスローされる前の状態を格納できる。
　　Thread.currentThread().setName(Context + TID + Params + current Time, ...);
  　結果：Queue Processing Thread, MessageID: AB5CAD, type: AnalyzeGraph, queue: ACTIVE_PROD, Transaction_ID: 5678956, Start Time: 02/04/2015 17:37

問題3：プロセスがハングしているがその理由を知らない。
　・任意のJavaプロセスに接続し、PIDを指すだけで、スタックトレース、現在保持しているすべてのスレッド、それらが保持しているロック、その他のメタデータを含む出力でソートされます。

問題4：上記の解決策はすべてコードの変更が必要。
　・アプリケーションに変更を加えるのは難しい作業。
　・BTraceを使用すると稼働中のJVMに接続してあらゆる情報を抽出することができる。

問題5：エラーの根本的原因に関するデータがない
　・ネイティブエージェントはJVMの低レベルのAPIセットにアクセスできる。
　・ネイティブエージェントには「Takipi」のようなものがある。
